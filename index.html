Three board types

One canvas. Three ways to ship.

Every Canflow board runs the same interface: cards, columns, drag. What changes is what each one knows. Pick the type that matches the work.

01 · Beta testing board

Where bugs, testers, and agents meet.

Ship a build, invite testers by email to any phase, and wire in Claude Code or Codex through MCP. Your agents pick up the same cards testers file, and drag them across the board themselves.

  • Email invites to any phase: testers, QA leads, or read-only stakeholders.
  • Claude Code & Codex via MCP: agents move cards as they work.
  • Screenshots · intensity · repro attached to every bug automatically.

02 · Roadmap board

Your users decide what ships next.

Turn any board into a public roadmap with one click. Anyone with the link can upvote what they actually want. No signup, no login. Ideas auto-sort to the top of Planned by real demand.

  • Public upvotes without a login: one shareable URL, real user demand.
  • Auto-sort by vote count so the roadmap reflects the crowd, not the founder.
  • Drop it in a tweet or a footer. Every visit is free market research.
